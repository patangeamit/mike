# ML CI/CD

**Summary**: ML CI/CD applies continuous integration and continuous deployment ideas to machine learning systems, with added concerns around data, models, retraining, and orchestration. The source treats it as a core automation layer in MLOps. 

**Sources**: `raw/The Ultimate MLOps Roadmap for 2026 A Complete Guide to Becoming an MLOps Engineer.md`

**Last updated**: 2026-04-19

---

The source says MLOps relies on CI/CD in a way that is similar to DevOps, but adapted to data, experiments, models, and ML pipelines (source: The Ultimate MLOps Roadmap for 2026 A Complete Guide to Becoming an MLOps Engineer.md). It includes GitHub Actions and Jenkins as example tools for building automated workflows that run tests, track experiments, retrain models, and trigger deployments (source: The Ultimate MLOps Roadmap for 2026 A Complete Guide to Becoming an MLOps Engineer.md).

## Automation Scope

The article describes ML CI/CD as covering automated model training, workflow orchestration, and model packaging with Docker (source: The Ultimate MLOps Roadmap for 2026 A Complete Guide to Becoming an MLOps Engineer.md). It mentions Airflow and Prefect for orchestration and frames the goal as automating the ML lifecycle so models move from development to production quickly and repeatably (source: The Ultimate MLOps Roadmap for 2026 A Complete Guide to Becoming an MLOps Engineer.md).
